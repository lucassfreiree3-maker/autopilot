Release Pipeline (apply-source-change) — 7 Stages

1. Setup
Read workspace config, resolve inputs, decode base64 content
GitHub Actions workspace.json
1.5 Session Guard
Acquire multi-agent lock — blocks if another agent is active
session-guard.yml Agent Lock
2. Apply & Push
Clone corporate repo → Apply code change → Fix lint → Push
git clone/push Claude / Codex LangChain
3. CI Gate
Wait for CI completion + smart comparison (pre-existing failure detection)
CI Workflows analyze-ci-failure CI failure alert
4. Promote
Update CAP/deploy values.yaml with new image tag
CAP repo Release notify
5. Save State
Record release-state.json on autopilot-state branch
autopilot-state
6. Audit
Immutable audit trail + release session lock
audit/*.json Agent ID
